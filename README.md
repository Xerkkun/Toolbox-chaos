# Chaos Toolbox

Interactive Windows desktop toolbox for exploring nonlinear dynamical systems and chaos theory.

## Included systems

- Continuous flows: Lorenz, Rossler, Chua, Chen, Lu, Duffing-Ueda, Rabinovich-Fabrikant, Rikitake, Sprott A, Thomas, Hindmarsh-Rose.
- Discrete maps: Henon, logistic, Ikeda.
- Other models: Mackey-Glass delay equation and Lorenz-96 high-dimensional ring model.

The app shows trajectory projections, time series, 3D attractor views, bifurcation diagrams, equilibrium/eigenvalue views, and attraction-basin grids where the model type supports them. Implemented systems run the heavy trajectory, bifurcation, and attraction-basin calculations through the native C backend.

## Installation for end users

End users should install **Chaos Toolbox** from the Windows installer generated by the project:

```text
installer/ChaosToolboxSetup-0.1.0.exe
```

After installation, launch the application from the Start Menu, the optional desktop shortcut, or:

```text
Chaos Toolbox.exe
```

The installed application is self-contained. End users do not need to install Python, PyQt6, NumPy, Matplotlib, pyqtgraph, gcc, or MinGW.

## Development

Development still uses the existing Python workflow. From PowerShell:

```powershell
.\run.ps1
```

Limit worker processes on Windows:

```powershell
.\run.ps1 -Workers 4
```

Linux and macOS helper scripts are also kept for development:

```bash
chmod +x run-linux.sh
./run-linux.sh
```

```bash
chmod +x run-macos.command
./run-macos.command
```

Install development dependencies manually if needed:

```powershell
python -m pip install -r requirements.txt
```

Run the smoke test:

```powershell
python scripts\smoke_test.py
```

## Build the Windows executable

The build machine needs Python and a C compiler such as gcc/clang. These are build-time tools only; they are not required on the end-user machine.

From PowerShell:

```powershell
.\packaging\windows\build.ps1
```

The script creates or reuses `.venv`, installs `requirements-build.txt`, compiles:

```text
core/bin/chaos_core.dll
```

verifies the native backend, and runs PyInstaller in windowed mode. The generated executable is:

```text
dist/Chaos Toolbox/Chaos Toolbox.exe
```

The reproducible PyInstaller spec is:

```text
packaging/pyinstaller/chaos_toolbox.spec
```

## Build the Windows installer

Install Inno Setup on the build machine, build the executable first, then compile:

```text
packaging/windows/ChaosToolbox.iss
```

For example, from the Inno Setup Compiler GUI open that `.iss` file and press **Compile**. The generated installer is:

```text
installer/ChaosToolboxSetup-0.1.0.exe
```

The installer places `dist/Chaos Toolbox` under Program Files, creates a Start Menu shortcut, offers an optional desktop shortcut, and includes the standard uninstaller.

## Project structure

- `main.py`: application entry point.
- `ui/`: PyQt6 interface, canvases, widgets, and rendering helpers.
- `core/`: numerical routines and native backend integration.
- `core/csrc/chaos_core.c`: native C backend source.
- `core/bin/chaos_core.dll`: compiled native backend used by Windows builds.
- `assets/`: educational/reference resources, including `chaos_dictionary.pdf`.
- `packaging/`: PyInstaller and Inno Setup packaging files.
